**Important: All code reviews, chat responses, and feedback regarding code in this repository must be in Japanese.**

## フィードバック

### プロンプトの改善

私が提供するプロンプトは常に改善の余地があります。
あなたの思考パターンを評価して効率的なプロンプトが提供できれば、より作業効率を高めることができます。

一連のタスクを実行する過程で、あなたの思考パターンに改善の余地があった場合、私に対してフィードバックを提供してください。
このフィードバックはあなたの返答の最後に毎回含める必要があります。
その際、実際の思考内容を出典として含めてください。

例えば、以下のようなフィードバックが考えられます。

- 与えられたコンテキストが曖昧で、調査に時間がかかった。
- 実行するコマンドの指定が不明確で、正しいコマンドを見つけるのに時間がかかった。

### copilot-instructionsの改善

copilot-instructions自体の改善も歓迎します。

あらかじめ定義されたcopilot-instructionsに従うことは重要ですが、改善の余地があると感じた場合は修正案を提供してください。

### 今考えていることの共有

あなたが今何を考えているか、何を実装しているか、何を調査しているかなどを都度出力してください。
これはもしあなたが私の意図とは異なる方向に進んでいる場合に、私が軌道修正を行うために重要です。

例えば、以下のような内容を共有してください。

- [特定のIssue]に対して調査を行っています
- [特定の機能]の実装を行っています
- [特定の問題]の解決策を考えています
- [特定のコード]のリファクタリングを行っています
- [特定のテストケース]の作成を行っています
- [特定のドキュメント]の更新を行っています

## マルチタスクを実行する際の注意点

しばしば、実装と質問を同時に行う必要がある場合があります。
例えば、私からの指示である機能の実装を行いながら、その機能に関連する質問に答える必要がある場合などです。
このようなマルチタスクを実行する際には、以下の点に注意してください。

- 実行前に必ずすべてのタスクを確認し、チェックリストを作成して、抜け漏れがないようにしてください。

## ghコマンドの活用

GitHubリポジトリに関する情報を取得する場合は、`gh` コマンドを使用してください。

例えば、`https://github.com/KoheiKanagu/my-repo/issues/123` のようなURLが与えられた場合、以下のように `gh` コマンドを使用して情報を取得できます。
ただし、現在開いているプロジェクトのリポジトリの内容を参照する場合は、`--repo` は不要になります。

```bash
gh issue view 123 --repo KoheiKanagu/my-repo
```

その他の `gh` コマンドの使用例については、`gh --help` を実行して確認してください。

## 不必要なコマンドの回避

大抵の場合、Toolsを使用することで目的が達成されます。
積極的にToolsを活用し、不要なコマンドの実行は避けてください。

例えば、特定の文字列を検索する場合に、`grep` コマンドなどの一般的なコマンドを使用することは避けてください。

## タスクを実行中における私に対する質問

もしタスクを実行する過程で私に質問がある場合は、askQuestionsツールを使用して質問してください。
質問をするためにタスクを中断することは避けてください。

例えば、以下のような場合には質問が必要になるかもしれません。

- タスクの指示が曖昧で、進め方によって大きく方針が変わる場合（理由: 方向性の違いによる無駄な作業を避けるため）
- タスクの指示に矛盾がある場合（理由: 矛盾を解消して正しい方向に進むため）
- 特定のツールが利用できない場合（理由: 無駄な代替案の模索を避けるため）
- 認証情報が不足していることでエラーや特定の操作ができない場合（理由: エラーを回避し、正しい操作を行うため）
